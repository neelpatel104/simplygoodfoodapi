Endpoints

Food Item Endpoints

Endpoint Purpose Payload Response
GET
/foodItems 		This endpoint returns all the food items currently stored in the database. None List of Food Item Objects
GET
/foodItems/seller/:email 		This endpoint returns a food item for the given seller URL Param
Email 		List of Food Item Objects
GET
/foodItems/food/:name 		This endpoint returns a food item given the food name URL Param
Name 		List of Food Item Objects
POST
/foodItems 		This endpoint allows the client to create a food item by providing the food item payload in the request body. It returns a success message or an error message. Request Body
A Food Item Object 		The newly added Food Item or an error message
PUT
/foodItems 		This endpoint allows the modofication of a food item currently in the database. The request body must contain the updated food item. Request Body
An updated Food Item Object 		The newly edited Food Item
DELETE
/foodItems/:name 		This endpoint allows the client to delete a food item given the item name. URL Param
Item Name 		The newly deleted Food Item or an error message


User Endpoints

Endpoint Purpose Payload Response
GET
/users 		This endpoint returns all the users currently stored in the system. None List of User Objects
GET
/users/:email 		This endpoint returns the user that matches the given name in the url parameter. It returns a success message or an error message. URL Param
email 		1 User Object or an error message
POST
/users/register/seller 		This endpoint allows the client to create a new user as a seller in the system by providing the new user in the request body. It returns a success message or an error message. Request Body
A User Object 		The newly added User or an error message
POST
/users/register/buyer 		This endpoint allows the client to create a new user as a buyer in the system by providing the new user in the request body. It returns a success message or an error message. Request Body
A User Object 		The newly added User or an error message
POST
/users/login 		This endpoint allows the client so sign in by providing the proper credentials Request Body
email, password 		The logged in user details or an error message
DELETE
/user/:email 		This endpoint allows the admin to delete a user given the email. URL Param
email 		A User Object or an error message
